## 📝 三个需要改的地方

所有配置都在 `picker.py` 文件的底部：

| 配置项 | 行号 | 频率 | 说明 |
|-------|------|------|------|
| **MY_CURRENT_PORTFOLIO** | 4920 | **每月必改** | 你的当前持仓（10 只股票的权重） |
| **MODE** | 4904 | 按需改 | 运行模式："pick"(快) / "backtest"(回测) / "both"(两者) |
| **EMAIL_CONFIG** | 4940 | 一次配置 | 邮件地址（用于接收月度报告） |

### 示例：更新持仓（每月最重要的一步）

```python
# picker.py 第 4920 行

MY_CURRENT_PORTFOLIO = {
    "BMO.TO": 0.152,        # Wealths imple 显示 15.2%  →  0.152
    "MFC.TO": 0.148,        # 14.8%  →  0.148
    "NTR.TO": 0.105,        # 10.5%  →  0.105
    # ... 共 10 只股票
}

# 规则：百分比 ÷ 100 = 权重
# 例：15.2% ÷ 100 = 0.152 ✓
```

## 🚨 常见问题

### Q1：收不到邮件？

**检查清单**：
1. 确认 `EMAIL_CONFIG` 的 `to` 邮箱地址正确
2. 检查垃圾邮件文件夹（Hotmail/Outlook 最常见）
3. 确认 Gmail app password 没过期（有效期 3 个月）
4. 检查网络连接（VPN/防火墙可能阻止 SMTP）

### Q2：权重和不等于 1.0？

**正常现象**：
- Wealthsimple 百分比有舍入误差
- 账户里有现金未投（0.95-1.05 都可接受）
- 系统会自动处理

### Q3：运行显示"KeyError: XIU.TO"？

**解决**：
```bash
# 清除数据缓存，重新下载
rm -rf ./simfin_data
python picker.py
```

### Q4：我想跑历史回测？

**改这一行**：
```python
# picker.py 第 4904 行
MODE = "backtest"    # 改成这个（需要 25-30 分钟）

# 然后运行
python picker.py
```

### Q5：模型预测全是 NaN？

**原因**：某只股票财报数据缺失

**解决**：
```bash
# 强制更新缓存
rm -rf ./simfin_data/fmp_cache.pkl
python picker.py
```

## 💡 最佳实践

### ✅ 做这些

1. **定时更新持仓**  
   每月 25-28 日之间更新一次，准确记录权重

2. **严格执行建议**  
   不要"凭感觉"加码或减码，相信模型的 18 个月回测

3. **备份报告**  
   每月的 PDF 保存到本地，用于年底对账

4. **周末跑回测**  
   想验证系统？MODE="backtest" 在周末运行（30 分钟）

### ❌ 不要这些

1. **频繁运行系统**  
   一周最多一次（数据变化会改预测）

2. **修改有风险的配置**  
   CONSTRAINTS、特征工程、模型参数都已优化

3. **忽视缓冲带**  
   系统设计了 ±12 档缓冲防止过度换仓（会浪费手续费）

4. **关注短期波动**  
   这是 6-12 个月的策略，别看 1 周收益就惊慌
